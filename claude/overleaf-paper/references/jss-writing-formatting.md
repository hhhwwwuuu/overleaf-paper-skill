# Journal of Systems and Software Writing and Formatting Reference

Source:

- https://www.sciencedirect.com/journal/journal-of-systems-and-software/publish/guide-for-authors

Use this as an operational checklist, not a replacement for checking the current Guide for Authors before submission.

## File Format and LaTeX

- Use editable manuscript source files. For LaTeX, prefer Elsevier's LaTeX template and keep source compilable.
- Keep figures, tables, supplementary files, and bibliography references organized with logical names.
- Do not submit equations or tables as images.

## Title Page

Check for:

- concise and informative title;
- author names and affiliations;
- corresponding author details;
- present/permanent address notes when needed;
- declarations required by the submission system.

## Abstract and Keywords

- Abstract should be self-contained and not numbered as a manuscript section.
- Avoid undefined abbreviations and citations in the abstract unless unavoidable.
- Provide 1 to 7 keywords.

## Highlights

- Provide 3 to 5 highlights.
- Each highlight must be at most 85 characters including spaces.
- Highlights should state concrete findings or contributions, not generic aims.

## Article Structure

- Use numbered sections and subsections in the main text.
- Typical empirical software engineering structure: Introduction, Background/Related Work, Methodology, Results, Discussion, Threats to Validity, Conclusion.
- Put acknowledgements before references.
- Keep appendices clearly labeled when present.

## Figures, Tables, and Equations

- Cite every figure and table in the manuscript text.
- Number figures and tables consecutively in order of appearance.
- Provide captions that define abbreviations and explain symbols.
- Tables must be editable text; avoid vertical rules and unnecessary shading.
- Equations must be editable text, numbered consecutively when displayed and referenced.

## References and Data

- Ensure every citation has a bibliography entry and every bibliography entry is cited, unless the style intentionally permits otherwise.
- Include research data and data availability statements when applicable.
- Do not invent funding, ethics, competing-interest, or CRediT contribution statements. Ask the user when information is missing.

## Length Awareness

- JSS encourages concise manuscripts; flag manuscripts that appear to exceed the guide's stated page expectations so the user can decide whether to shorten or justify the length.
